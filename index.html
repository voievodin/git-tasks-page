Tasks

According to your number in the group list you need to write an instruction in a form of set of commands that solves the corresponding problem (the one assigned to you) in the list below.

  1. The branch b contains 3 commits (comparing to the master branch). Write a set of commands that will make a single commit out of those 3 and merge the b branch to the master branch. Remove the b branch after merge is done.

    Hint

    There are multiple ways to achieve it. A simple one would be to use soft reset and commit with --amend.
  2. The branch a contains 2 commits (comparing to the master branch). Write a set of commands that will do the following: change the first commit to contain a new file named cf1.txt with arbitrary content; change the second commit to contain a new file named cf2.txt with arbitrary content. Ensure that only 2 commits remain in the branch a (comparing to the master branch). In linux (or gitbash in windows) you can use touch command to create a file, for example, touch cf1.txt. In case interactive operation is used use [action] square brackets to point what should be done in the interactive mode. For example, in order to accept suggested interactive rebase changes use [accept] comment, like rebase -i HASH // [accept].

    Hint

    You can use git commit --fixup HASH and git rebase -i --autosquash HASH commands to 'tweak' existing commits in a convenient way.
  3. Roll back the change that the commit #3 introduced. Do that in the branch named rollback_commit_3. Merge the change to the master branch. Remove the newly created branch.

    Hint

    git revert HASH can be used to reverse the change applied by the commit.
  4. Move the changes in commits 7 and 8 to the branch b, as result the branch b should contain 2 more commits (comparing to the master branch). Merge the branch b to the master branch. Remove both a and b branches.

    Hint

    The simplest way to achieve it would be using the git cherry-pick HASH command
  5. Create a new branch based on commit 3. Add a new file named cf1.txt. Commit the file. Change the base commit of the branch from 3 to 9. In linux (or gitbash in windows) you can use touch command to create a file, for example, touch cf1.txt.

    Hint

    git rebase HASH will do the trick
